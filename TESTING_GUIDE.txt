------------------------------------

Expected:

Each model includes a `metrics` object with training/evaluation output, for example:
- Classification (hold-out test): auc, accuracy, precision, recall, f1, threshold
- Validation tuning: val_accuracy, val_f1, val_profit, val_sharpe
- Simulated trading on test split: trades, gross_pnl, win_rate, avg_trade_return, max_drawdown_like

- Dataset: n_samples, n_symbols, pos_rate_test, majority_accuracy, label_method, selected, dataset_path

UI: open the app → Models page → **Model evaluation** shows the same fields in tables;
click a row in "Model registry" to switch models; use "Show full metrics JSON" for the raw object.

Expected:

4) Train a New Model with Live Sentiment
----------------------------------------
In Swagger:
- POST /api/ml/train/xgb

Recommended body:
{
  "universe": "custom",
  "symbols": ["BTCUSDT", "ETHUSDT", "SOLUSDT", "XRPUSDT", "DOGEUSDT"],
  "interval": "1m",
  "limit_per_symbol": 350,
  "tune": false,
  "optimize_metric": "f1",
  "label_horizon": 1,
  "label_threshold": 0.0,
  "sentiment_post_limit": 60,
  "sentiment_required": false,
  "min_sentiment_coverage": 0.0
}

Then:
1. Copy returned job_id
2. Poll GET /api/ml/jobs/{job_id} until status=succeeded
3. Open GET /api/ml/models and verify newest model
4. Check metrics.sentiment_coverage_mean > 0 for sentiment usage

Notes:
- strict sentiment_required=true may fail if Reddit mentions are sparse
- use false for reliability, then validate sentiment_coverage_mean


4b) Large-scale training (~1.2M balanced, default in API)
---------------------------------------------------------
The backend now **paginates Binance klines** when `limit_per_symbol` > 1000 (API max per call).

Defaults (POST /api/ml/train/xgb):
- `limit_per_symbol`: 120000  → 10 top coins ≈ 1.2M raw rows before balancing
- `balance_classes`: true
- `balance_per_class`: 600000 → up to **600k samples per class** (~1.2M total), oversampling if needed
- `optimize_metric`: roc_auc (better ranking on balanced data than accuracy alone)
- `tune_trials`: 40; large training sets use a wider hyperparameter search
- **Market + news together:** Binance OHLCV/technical features are always used. **Reddit sentiment**
  is included when `sentiment_post_limit` > 0 (default **150**). Set `sentiment_post_limit` to **0**
  to train on technical data only (faster). Requires `REDDIT_CLIENT_ID` / `REDDIT_CLIENT_SECRET` /
  `REDDIT_USER_AGENT` in the backend environment; after merge, the last known sentiment is **forward-filled**
  along the 1m timeline so sparse posts still influence most rows.
- Check `metrics.sentiment_coverage_mean` after training — should be > 0 when Reddit data matched your window.

Expect long runtimes (many minutes) and large CSVs under `CRYPTOVOLT_DATA_DIR`. The UI “Train + Auto-tune” uses these defaults.

5b) Model evaluation (registry metrics)
---------------------------------------
After training, evaluation is stored on each model as `metrics` (see GET /api/ml/models).
On the frontend **Models** page, open **“Model evaluation”** — tables group the same fields.

How to read each block:

**Classification (hold-out test)** — Static test split; not used for threshold tuning.
- **AUC** — Rank quality of predicted probabilities (0.5 = random, 1 = perfect ranking).
- **Accuracy / Precision / Recall / F1** — Standard scores at the chosen **decision threshold**.
- **Decision threshold** — Probability cutoff for “buy/long” vs not (tuned on validation when `tune=true`).

**Validation (threshold search)** — Metrics when scanning thresholds on the **validation** slice only.
- **Val accuracy / Val F1** — Best achievable at some threshold on val data.
- **Val profit / Val Sharpe** — Simple simulated long-only PnL / Sharpe on val forward returns (fees in `label_cost_bps`).

**Simulated trading (test split)** — Applies the final threshold to **test** predictions + forward returns.
- **Gross PnL / trades / win rate** — Naive sum of returns where model says long (minus fee); use for relative comparison between runs, not live guarantees.
- **Max drawdown-like** — Approximate worst peak-to-trough on that simulated equity curve.

**Walk-forward estimate** — Several time-ordered folds; retrains on past, scores future chunk.
- **WF avg PnL / WF avg win rate** — Less optimistic than a single split; better sanity check for overfitting.

**Dataset & labels**
- **n_samples / n_symbols** — Scale of training data.
- **pos_rate_test / majority_accuracy** — Class balance; high accuracy can be misleading if labels are skewed.
- **label_method** — `simple` | `vol_cost` | `triple_barrier` (trading-aware labels).
- **selected** — `baseline` vs `tuned` (whether hyperparameter search beat baseline).

**API (same data):**
  curl -s http://localhost:8000/api/ml/models
  # Inspect each object: metrics.{auc,f1,val_sharpe,wf_avg_pnl,...}

**UI checklist:**
1. Models page → confirm **Model evaluation** card lists metrics (not empty).
2. Click different rows in **Model registry** to compare versions.
3. Use **Show full metrics JSON** for exact numbers / `tuning` details.
4. After a job finishes, **Training job** also shows the same evaluation tables.

7) Automation (Paper Trading) Tests
-----------------------------------
1. Start from Dashboard automation controls (or API):
   POST /api/trading/automation/start
2. Suggested config:
   - interval: 1m
   - stop_loss_bps: 250
   - take_profit_bps: 400
   - use_proba_thresholds: true

4. Stop automation:
   POST /api/trading/automation/stop

Expected:
- simulated BUY/SELL actions occur
- risk exits (stop/take profit) can force SELL

10) Troubleshooting
-------------------
A) API unreachable
- Check Docker Desktop is running
- docker compose ps
- docker compose up -d backend frontend trainer

B) "Unable to connect to remote server"
- backend restarting/crashed; wait and retry
- check backend logs:
  docker compose logs backend --since 10m

C) Training too slow
- reduce symbols
- reduce limit_per_symbol
- set tune=false
- lower sentiment_post_limit

D) No usable symbol data fetched
- if sentiment_required=true, switch to false
- expand symbols to high-mention coins (BTC/ETH/SOL/XRP/DOGE)

E) Sentiment coverage remains zero
- verify Reddit keys in .env
- restart backend after .env change
- confirm model metrics.sentiment_coverage_mean
